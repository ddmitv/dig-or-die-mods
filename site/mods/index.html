📖 Detailed Guides & Troubleshooting

📋 Manual Installation Guide

▼
Prefer a visual guide? Check out the Community Steam Guide for an alternative step-by-step walkthrough with screenshots.
1

Download BepInEx

Get the 32-bit (x86) Windows BepInEx version 5.4.23 or newer from the BepInEx GitHub releases.

The file name should look like BepInEx_win_x86_5.4.23.3.zip
2

Install BepInEx

Extract the downloaded .zip contents into your game directory where DigOrDie.exe is located.

In Steam: right-click the game, select Manage → Browse local files. Optionally, back up your game by copying the entire Dig Or Die folder elsewhere before extracting.
After extraction, your game folder should look like this:
Dig or Die/
DigOrDie.exe
DigOrDie_Data/
winhttp.dll new
doorstop_config.ini new
.doorstop_version new
changelog.txt new
BepInEx/ new
config/
core/
patchers/
plugins/ ← put .dll files here
3

First Launch

Run the game through Steam. Directly running DigOrDie.exe won't work.

The game will always show a black screen on first launch — don't wait for it to start. Wait until the BepInEx/config folder appears, then close the game.
4

BepInEx Configuration

Open BepInEx/config/BepInEx.cfg in a text editor and find near the end:

[Preloader.Entrypoint] ## The local filename of the assembly to target. # Setting type: String # Default value: UnityEngine.dll Assembly = UnityEngine.dll ## The name of the type in the entrypoint assembly to search for the entrypoint method. # Setting type: String # Default value: Application Type = Application ## The name of the method in the specified entrypoint assembly and type to hook and load Chainloader from. # Setting type: String # Default value: .cctor Method = .cctor
Change Type = Application → Type = MonoBehaviour
Optional: Enable debug console

Find the [Logging.Console] section in BepInEx.cfg and set Enabled = true. A console window with logging will appear on game start. 

[Logging.Console]
## Enables showing a console for log output.
# Setting type: Boolean
# Default value: false
Enabled = true
5

Install Plugins

Download plugin .dll files from below and place them into the Dig or Die/BepInEx/plugins folder.

6

Play!

Run the game through Steam. Plugins should now be active.

⚙️ Plugin Configuration

▼
  • Many plugins have configuration files under BepInEx/config. A config file is created automatically the first time the game is started with that plugin.
  • Usually, config changes require a game restart to take effect, though some plugins read settings dynamically (check the plugin's documentation).
  • To reset a plugin's config to defaults, delete its config file — it will be recreated on next launch.
Example config file structure after installing some plugins:
Dig or Die/BepInEx/config/
BepInEx.cfg ← main BepInEx config
com.example.precise-clock.cfg
com.example.better-chat.cfg
com.example.autosave-tweaks.cfg
Config files are named after the plugin's GUID and appear after first launch
Use Configuration Manager (see Recommended Plugins below) to edit configs in-game via a GUI.

🔧 Troubleshooting

▼

Check Dig or Die/DigOrDie_Data/output_log.txt for errors. Visit the BepInEx Troubleshooting page for more help.

🗑️ Uninstallation

▼
  • Disable specific plugin: Delete its .dll from BepInEx/plugins.
  • Temporarily disable all mods: Rename winhttp.dll to winhttp.dll.off. Rename back to re-enable.
  • Full uninstall: Delete BepInEx/, winhttp.dll, doorstop_config.ini, .doorstop_version, changelog.txt. Then verify integrity of game files through Steam.

🛠️ Building Plugins Yourself

Compile the plugins from source code

Prerequisites

Install .NET SDK (should support C# 12.0).

Set the environment variable DIG_OR_DIE_PATH to your Dig or Die installation path to automatically deploy built plugins to BepInEx\plugins.

Using Visual Studio

You can download Visual Studio from microsoft.com.

  • Open the plugins/ directory (where all plugin source code is located).
  • The plugin's directory should contain a .csproj file that you need to open with Visual Studio.
  • You can build the current plugin with Ctrl+B or Build → {plugin name}.

Using Command Line

Open a terminal in the repository root and run:

dotnet build plugins/{plugin name}
Add -c Release to build with Release configuration.
